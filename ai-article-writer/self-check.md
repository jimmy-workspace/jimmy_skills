# 终检清单

写完文章后，逐条过一遍。全部打勾才能发布。

## 标题

- [ ] 字数在 15–40 之间
- [ ] 包含数字、情绪词、时效词中的至少两项
- [ ] 前半句有情绪/冲突，后半句有信息/结论
- [ ] 不是纯客观描述，有明确的情绪倾向

## 开头

- [ ] 第一句话直接下判断/抛悬念，没有背景铺垫
- [ ] 前 3 段总共不超过 200 字
- [ ] 明确告诉读者"读完你能拿走什么"

## 结构

- [ ] 正文分了 3–7 个编号模块
- [ ] 每个模块有加粗小标题
- [ ] 逻辑是递进的（现象 → 分析 → 观点 → 建议）

## 论证

- [ ] 每个观点后面都有具体例子 / 数据 / 实测
- [ ] 没有连续 3 段以上的抽象论述
- [ ] 复杂概念都用类比或人话翻译过

## 语言

- [ ] 通读一遍像在和朋友聊天
- [ ] "我""你"交替出现，有对话感
- [ ] 至少命中 5 个标志性词汇（说实话/你没听错/说白了/有一说一等）
- [ ] 没有官方通稿式的表达
- [ ] 没有"适当""合理""有时"等模糊词

## 情绪

- [ ] 开头有明确的情绪定调（震撼/兴奋/焦虑/感动）
- [ ] 60–80% 处有一次从"具体工具"到"时代意义"的跃迁
- [ ] 结尾有人文关怀或积极愿景

## 结尾

- [ ] 有"写在最后"模块
- [ ] 总结了核心价值（1–2 句）
- [ ] 给出了 1–2 个可立即执行的行动建议
- [ ] 最后一句话有感染力（金句/愿景/断言）
- [ ] 有标准 CTA 三连

## 排版

- [ ] 90% 段落不超过 2 句话
- [ ] 没有超过 3 行未拆分的段落
- [ ] 核心观点 / 产品名 / 步骤已加粗
- [ ] 手机端浏览有"呼吸感"
- [ ] 每出现长段解释后跟了截图 / 例子 / 清单

## 终极检验

- [ ] 全文朗读一遍，口语化流畅不卡壳
- [ ] 所有"我认为"都有"我体验到"作为支撑
- [ ] 标题、开头、结尾形成闭环，共同强化核心价值
- [ ] 一个不懂技术的朋友也能读完觉得"有用"或"有意思"
