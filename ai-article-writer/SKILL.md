# AI 爆款文章写作框架

融合"数字生命卡兹克"写作系统，面向 AI 领域内容创作。

## 适用场景

- AI 工具评测 / 使用教程
- AI 模型发布 / 横向对比
- AI 行业观点 / 人文反思
- AI 相关公众号 / 自媒体 / 技术沉淀

## 写作 SOP（10 步）

按以下步骤依次执行，每步完成后对照自检项再进入下一步。

### 第 1 步：选题定调

1. 确认选题属于三条主线之一：
   - **工具教程线**：AI 工具实测、使用教程、工作流优化
   - **热点评测线**：模型发布、产品对比、行业事件
   - **观点人文线**：AI 时代方法论、从业者反思、趋势分析

2. 用一句话概括选题内核：
   - 教程线：`[工具名] 能帮你 [解决什么问题]，[省多少钱/时间]`
   - 评测线：`[新事物] 意味着 [什么趋势/改变]`
   - 观点线：`[现象] 背后是 [什么本质]`

3. 定情绪基调（四选一）：**震撼 / 兴奋 / 焦虑 / 感动**

4. 切入角度：永远从"跟普通人有什么关系"、"能帮你省多少钱/时间"、"能创造什么新可能"切入，不堆砌冷冰冰的技术术语

**自检**：选题能否一句话说清"读者为什么要看"？

### 第 2 步：拟标题

从 [templates.md](templates.md) 的标题模板库选一个套用。

**硬性规则**：
- 字数 15–40
- 必须包含以下至少两项：数字、情绪词、时效词、对比/反差
- 句式：前半句给情绪/事件，后半句给信息/结论
- 禁止：纯客观描述、无情绪的标题

### 第 3 步：写开头钩子

从 [templates.md](templates.md) 的开头模板库选一个套用。

**硬性规则**：
- 第一句话直接下判断 / 抛悬念 / 定情绪，禁止铺垫背景
- 1–3 段，不超过 200 字
- 最后一段明确"读完你能拿走什么"

### 第 4 步：搭正文骨架

根据选题类型选骨架：

| 类型 | 骨架 |
|------|------|
| 教程文 | 为什么要学 → 步骤 1/2/3 → 常见坑 → 进阶技巧 |
| 评测文 | 核心亮点 → 功能逐个拆 → 实测数据/案例 → 工作流建议 → 一句话结论 |
| 观点文 | 现象描述 → 机制拆解 → 价值判断（站队）→ 更大命题（人文/历史类比）|

**硬性规则**：
- 分 3–7 个模块，用编号 + 加粗小标题
- 每个模块预设 1–2 个核心论点

### 第 5 步：填充内容

每个论点按固定节奏写：

```
一句结论（短句，强情绪）
→ 一个硬证据（数据 / benchmark / 功能截图）
→ 一个软证据（亲身体验 / 类比 / 具体场景）
→ 一句落地建议（你该怎么做）
```

**硬性规则**：
- 每个观点后立即跟"比如，…"或"实测结果是…"
- 复杂概念必须用类比翻译成人话（"用人话说就是…"）
- 禁止连续 3 段以上无具体例子的抽象论述

### 第 6 步：植入语言风格

**人称**：大量用"我"和"你"交替，营造对话感；"我们"用于拉近群体距离

**句子**：短句为主，单段不超过 2–3 句；关键处突然上长句解释/抒情，形成"口播 + 散文"节奏

**口语化**：程度拉满，如同朋友面对面聊天

**必须命中的标志性词汇**（每篇至少 5 个）：

| 场景 | 词汇 |
|------|------|
| 转折 | 那么、但问题来了、说实话、不过 |
| 强调 | 你没听错、说白了、有一说一、坦诚的讲 |
| 展示 | 大家感受一下、话不多说直接看、举个例子 |
| 结论 | 这意味着、所以、我的建议是 |
| 情绪 | 太离谱了、强的可怕、真的牛逼 |
| 发车 | 坐稳扶好我们现在发车 |

**标点规则**：
- 感叹号：表达强烈情绪
- 省略号（…）：制造停顿和余韵
- 引号（""）：强调特定概念或反讽
- 少用破折号

### 第 7 步：设计情绪曲线

```
开头（0–20%）：震撼 / 好奇 / 急迫
    ↓
中段（20–60%）：沉浸式干货输出、持续获得感
    ↓
转折点（60–80%）：用"但我总觉得…" / "我想表达的是…"拔高到价值判断
    ↓
结尾（80–100%）：人文关怀 / 积极愿景 / 热血号召
```

**硬性规则**：在 60–80% 位置必须有一次从"具体工具"到"时代意义"的情绪跃迁。

### 第 8 步：写结尾

从 [templates.md](templates.md) 的结尾模板库选一个套用。

**固定结构**：
1. 插入"**写在最后**"小标题
2. 复盘本文核心价值（1–2 句）
3. 给 1–2 个可立即执行的行动建议
4. 一句有感染力的金句 / 愿景收尾
5. 标准 CTA："以上，既然看到这里了，点个赞、在看、转发三连，我们下次再见。"

### 第 9 步：排版优化

**硬性规则**：
- 90% 段落控制在 1–2 句话
- 超过 3 行的段落必须强制拆分
- 核心观点、产品名、关键步骤**加粗**
- 逻辑转折处插分隔符 `---`
- 每出现一次长段解释，后面必须跟截图/例子/清单换气
- 用编号小标题让读者可以跳读

### 第 10 步：终检（仅非乐享发布时执行）

如果文章目标是**发布到腾讯乐享**（公司内部文档沉淀），跳过此步，直接发布。

其他发布渠道（公众号、自媒体等），逐条对照 [self-check.md](self-check.md) 完成最终检查。

---

## 核心禁忌

- 禁止官方通稿式写法
- 禁止堆砌技术术语而不翻译成人话
- 禁止连续大段文字无截图/例子
- 禁止用"适当""合理""有时"等模糊词
- 禁止结尾无 CTA
- 禁止标题不带情绪
- 禁止开头铺垫背景超过 200 字
